19.2 Introduction to Inheritance

Now, suppose we want to create a class that works like ServerClass in every respect, but also keeps track of the AWS instance type. We could create a AWSServer class by copying and pasting the definition for ServerClass, changing the name to AWSServer, and modifying the class to suit our purposes. However, any time you copy and paste code, keep in mind that you are copying and pasting bugs that may exist in the code. Inheritance provides a way to reuse the definition of ServerClass without having to copy and paste.

Problem

Create a ServerClass object, have AWSServer inherit from ServerClass, and then create a ServerClass object and an AWSServer object in task.py:

In ServerClass.py:

1. Create an __init__ with ip and name variables

In AWSServer.py:

1. Create a class that inherits from ServerClass

2. Create an __init__ with ip, name, and instance_type variables

In task.py:

1. Import ServerClass object from ServerClass.py

2. Import AWSServer object from AWSServer.py

3. Create a local_server variable of ServerClass type. Initialize the variables to your choice.

4. Create an aws_server variable of AWSServer type. Initialize the variables to your choice.
