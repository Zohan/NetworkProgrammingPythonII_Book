19.2 Introduction to Inheritance

Suppose we want to create a class that works like ServerClass in every respect, but also keeps track of the Operating System. We could create a ServerWithOperatingSystem class by copying and pasting the definition for ServerClass, changing the name to ServerWithOperatingSystem, and modifying the class to suit our purposes. However, any time you copy and paste code, keep in mind that you are copying and pasting bugs that may exist in the code. Inheritance provides a way to reuse the definition of ServerClass without having to copy and paste.

Problem

In task.py:

1. Create a ServerWithOperatingSystemClass object. Have ServerWithOperatingSystemClass inherit from ServerClass.

2. Add a constructor (hint: __init__) to ServerWithOperatingSystemClass that stores ip, name, and operating_system variables.

3. Add a get_inherited_server_class_information method to ServerWithOperatingSystemClass. It should call get_server_class_information in ServerClass and append OS information. Example: "Name: AzureVM IP: 36.63.12.45 OS: Windows"
