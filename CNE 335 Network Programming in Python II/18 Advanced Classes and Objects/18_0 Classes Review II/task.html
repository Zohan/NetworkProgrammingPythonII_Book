18.0 Review II: Objects as Arguments and Parameters (17.7), Instances as Return Values (17.9)

You can pass an object as an argument in the usual way. Functions and methods can return objects. This is actually nothing new since everything in Python is an object and we have been returning values for quite some time. The difference here is that we want to have the method create an object using the constructor and then return it as the value of the method.

Problem

You will create two classes, NetworkCardClass and NetworkServerClass.

In NetworkCardClass:

1. Add constructor that stores an ip_address (str) and a model_name (str)

In NetworkServerClass:

1. Add constructor that stores an ip_address (str) and a network_card (NetworkCardClass)

2. Add a method, get_server_details, that returns the system info as a string. Example: "Name: AWS EC2 IP: 36.63.23.21 Network Card: Intel Gigabit Adapter"
